# Meme Skill 模板

> 此模板定义了 meme skill 的 SKILL.md 编写规范。创建新梗 skill 时，严格按此结构填写。

## Frontmatter

```yaml
---
name: meme-<拼音>
description: <梗名>（<分类>）知识库。当用户提及"<关键词1>""<关键词2>"或相关梗时触发。涵盖起源、演变、用法、衍生概念等完整信息。
---
```

- `name`：`meme-<拼音>`，全小写，连字符分隔，拼音不带声调（如 `meme-niyijiku`）
- `description`：核心关键词 3-6 个，优先覆盖不同写法/别称；分类用 2-4 字概括

## 编写要点

1. **LLM 已知的知识从简**：如果 LLM 训练数据已覆盖该梗的基础信息（如角色设定），概述部分简化，重点放在最新的语义演化
2. **至少 2 个独立来源**：起源人物、时间线、核心含义需交叉验证
3. **时间格式统一**：使用 `YYYY-MM` 或 `YYYY-MM-DD`
4. **Agent 指南要具体**：不能只写「灵活回应」，要给出可操作的判断标准
5. **引用来源可追溯**：每个关键事实应有对应的参考资料链接
